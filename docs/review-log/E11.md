# E11 (ADR-0014 / D017–D019) — agent- & provider-agnostic tier routing (Ticket A)

**Change under review:** new `ports/agent_launch.py` (the `AgentLaunch` renderer seam +
the one shipped `OpencodeRenderer`), `api.py` (tier-vid pool path replacing the
`select_live_entry` role path; `_pool_routes`/`_http_probe`/`_acp_for_proxy`/`_split_model`
retired), `adapters/acp.py` (resolve the tier vid per-dispatch), and
`tests/test_agent_launch_routing.py` (the wire contract). The three lenses ADR-0014 names:

- **[Coupling-inventory completeness]** The ADR Consequences warn the opencode coupling is
  "broader than three functions." I relocated the FULL set behind the seam: the
  `OPENCODE_CONFIG_CONTENT` blob shape (`_acp_for_proxy`/`_start_proxy_acp`), `_split_model`,
  and the env contract (`_ACP_BASE_PASSTHROUGH`/`_ACP_KEY_PASSTHROUGH`/`_acp_passthrough_env`)
  — the last three now live in `ports/agent_launch.py` and `api.py` imports
  `_acp_passthrough_env` for the one no-proxy interim path. `_pool_routes` (D2 — unsorted, no
  cost rank) and `_http_probe` (only the retired select path used it) are DELETED, not moved.
  `_start_proxy_acp` keeps its single-upstream proxy setup but renders the launch through the
  seam, so no second copy of the opencode config shape survives in `api.py`. Verified by grep:
  no `_split_model`/`_pool_routes`/`_acp_for_proxy`/`_http_probe` references remain in `api.py`.

- **[Contract re-homing (B4)]** Retiring `select_live_entry` removed two load-bearing keys.
  Both are re-homed from the gateway's OWN observability, not dropped: (1) the dry-pool
  early-return still yields `{status:"exhausted", note:…}` (now keyed by the resolved tier
  vid) when `gateway.load_config` compiles an empty chain; (2) the `failover` result key is
  rebuilt by `_tier_failover_note(status_snapshot())` — served provider (`providers[*].served`)
  + skipped-provider list (`recent_failovers[*].failovers[*].provider`). A whole-chain 429 is
  relayed terminally and surfaces as `OutcomeStatus.EXHAUSTED` via the existing `acp.py:162-163`
  rate/quota mapping (unchanged). Tests pin all three: free-first wire order, the skipped+served
  note, the whole-pool-exhausted note, and the empty-note dry case.

- **[Over-claimed modularity — only opencode ships]** Exactly ONE renderer is written
  (`OpencodeRenderer`); `render()` defaults to it and exposes a `renderer=` injection point
  rather than branching on agent name. NO generic/`claude-code` renderer is shipped on spec
  (ADR-0014 D3/B1 + Risks: additional renderers gate on a live `charon doctor` probe). The
  test asserts the AGNOSTIC seam surface — `requested_model == tier_vid`, no key in the env,
  the proxy URL reaches the agent — never an opencode config shape, so any future renderer
  honoring the seam satisfies it. The "agent-agnostic" claim is scoped to the seam boundary
  (one proven renderer), matching the ADR's honest modularity framing.

D5 honored structurally: `AcpBackend.dispatch` now resolves `tier.value` → `last_tier_vid`
per dispatch (the param was previously ignored); Phase B (backend-selection-by-tier via
`router.route`) is left as the named extension point, not built here.

Privileged core stays stdlib-only (`json`/`os`/`shlex`/`dataclasses`); no new deps. Gate
green (pytest 536, ruff, mypy 45 files, boundary, version 0.2.0).
