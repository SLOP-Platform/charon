## 2026-06-26 — E8 — Opt-in batch-atomic auto-land (ADR-0012)

**Change under review:** building ADR-0007 D5 — an opt-in auto-land path on top of the
propose-default land gate (`land.py`) + its config surface (`config.py`). This is the
**highest-blast-radius** feature in Charon: it merges agent-produced code with no human at
the merge, and parallelism multiplies one ticket injection into N landings. Per the house
rule, three adversarial lenses were run against the design BEFORE code; reconciliation
below. Reviewers flag; the author reconciles against physics ([[charon-build-methodology]]).

- **[Lens 1 — privilege-escalation] A unit edits the repo to enable its own auto-land or
  widen its allowlist.** Charge: if the opt-in flag / allowlist / sensitive-set lived in the
  worktree, a single landed unit could flip auto-land on or grant itself new paths, then the
  *next* batch self-escalates. Reconciled (ADR-0012 D1/D3): the switch, allowlist, and
  sensitive-set are read **from outside the worktree** (engine config in `config_dir()` /
  `CHARON_AUTOLAND`), never from repo files. A file *named* like the config inside the diff is
  sensitive (D4) and holds. The config may only **extend** the sensitive set, never shrink it.

- **[Lens 2 — injection/provenance] A clean-looking, in-scope, test-passing but hostile unit
  rides the gate, and the merge executes worker-authored code.** Charge: the gate is an
  integrity check, not an adversary model; the ticket is the attack surface; `conftest.py` /
  `tests/` / build hooks execute *inside* the gate, so "run tests then auto-land" runs attacker
  code. Reconciled (ADR-0012 D4/D5/D3): sensitive paths (incl. all gate/CI/tooling-executed
  files) HOLD **always, even on green**; acceptance + tests run in a **disposable sandbox
  worktree**, never the live tree; and the **path-allowlist** shrinks the auto-landable surface
  to declared source dirs only. Residual trust is explicit and operator-granted; for untrusted
  ticket origins the guidance is to leave auto-land OFF (propose-only — ADR-0007 D4).

- **[Lens 3 — fail-open] A check silently does not run and the batch reads green.** Charge: a
  missing scanner, a timeout, an unreadable diff, a per-unit-green-but-integration-conflicting
  batch, or a partial half-land could each let bad code through or land an inconsistent set.
  Reconciled (ADR-0012 D2/D6/D7): scanners are **required/fail-closed** when auto-land is on
  (flip D007) — finding HOLDS, unavailable/timeout/error HOLDS; gitleaks is `expected` (missing
  HOLDS); **batch-atomic** integration on a throwaway branch with a single CAS `update-ref`
  means any partial/conflicting batch lands **NOTHING**; and the default of every decision
  branch is HOLD. A gate that cannot run never reads as green.

**Anti-dilution (ADR-0005 R3 / ADR-0007 D11):** the propose-default path is unchanged
(additive optional params with propose-preserving defaults); the gateway request path and
install footprint are untouched; privileged core stays stdlib-only (git + scanners via
subprocess, never imported).
