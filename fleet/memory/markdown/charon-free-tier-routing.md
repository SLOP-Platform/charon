**COST+QUALITY ROUTING PRINCIPLE (operator, 2026-07-04 — governs defaults + failover):** prioritize the most COST-EFFECTIVE model that does the job well; don't default to expensive-but-good models. Concretely: (1) free/very-cheap models that do a SPECIFIC task well should be the PRIMARY for that task — UNLESS they make too many costly mistakes (quality-aware; Charon has a `quality_scorer` to feed this back). (2) Expensive-but-excellent models (e.g. GPT-5.5) are TOO costly for day-to-day — restrict them to high-value work (review/consensus/final passes), not routine. So routing = cost-rank + capability/tier + quality-feedback, with premium models gated to premium roles. This is the north star for the auto-`cost_rank`/`cost_class` ticket (#6), the pools→tier simplification (#19), and tier-fallback. **v0.3.3 DEPLOYED (2026-07-04):** failover-on-unsupported-400 fix live on self-hosted-runner (build_sha ec0d8ae).
